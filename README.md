# hs

The Human Standard backbone for Claude Code. Governance, memory, and a session protocol — installed in one command.

Built on the [Human Standard](https://humanstandard.co) framework.

---

## Setup

```bash
git clone https://github.com/humanstandardsystems/hs.git
cd your-project
bash /path/to/hs/install.sh
```

Then open Claude Code in your project and run:

```
/standard
```

Claude will guide you through the governance questionnaire.

---

## What gets installed

**Per-project (in your repo)**
- `_governance/` — your AI's operating rules
- `/standard` slash command + startup hook
- `primer.md` template at the project root (session state across runs)

**User-level (in `~/.claude/`)**
- `/done` — end-of-session ritual: updates `primer.md` and runs plugin hooks
- Brain protocol: `brain.py` + `/decide`, `/todo`, `/done-todo`, `/glossary`, `/context`, `/sync` for shared knowledge across machines
- ICM memory hooks (auto-wired if [`icm`](https://github.com/rtk-ai/icm) is installed via brew)

---

## The four pillars

| Pillar | What it is |
|--------|-----------|
| **Governance** | Personal rules generated by `/standard` — foundation, enforcement, interpretation, policy. Loaded every session. |
| **Memory (ICM)** | Long-term semantic memory. Auto-stores decisions, errors, preferences. Recalled at the start of relevant tasks. |
| **Primer protocol** | A `primer.md` at every project root captures session state. `/done` updates it. Next session boots oriented. |
| **Plugin contract** | A spec (`docs/plugin-contract.md`) for extending `/rise` and `/done` with project-specific behavior. |

---

## Governance — what `/standard` generates

Four documents written to `_governance/`:

| Document | What it does |
|----------|-------------|
| `foundation.md` | Your core rules — what AI can and can't do, your cognitive sovereignty, your personal boundaries |
| `enforcement.md` | How violations are handled while you're in control |
| `interpretation.md` | Domain-by-domain permissions — business, personal, financial, creative |
| `policy.md` | What happens to your AI systems if something happens to you |

The questionnaire asks about your philosophy, fears, and desires — not technical preferences. The structure stays tight; your voice fills it in.

---

## Brain protocol (optional, per-project)

Add a single line to your project's `CLAUDE.md`:

```
brain: git@github.com:you/your-brain.git
```

Then `/decide`, `/todo`, etc. record knowledge as files. `/done` commits and pushes them — your brain travels between machines.
